Авторизация и аутентификация пользователя

Вопросы по лекции:

1. Что такое авторизация?

Авторизация — это процесс проверки прав пользователя на выполнение определенных действий или доступ к определенным ресурсам после того, как его личность была подтверждена (аутентифицирована).

2. Что такое аутентификация?

Аутентификация — это процесс проверки подлинности пользователя, подтверждение того, что он действительно является тем, за кого себя выдает (например, с помощью логина и пароля, отпечатка пальца, смарт-карты).

3. В чем разница между авторизацией и аутентификацией?

Аутентификация отвечает на вопрос "Кто вы?" (подтверждение личности), а авторизация — "Что вам разрешено делать?" (проверка прав доступа).

Пример: Вход в систему по паролю — это аутентификация. Доступ к определенным файлам после входа — это авторизация.

4. Как создать базу данных?

В SQL-серверах (например, MySQL, PostgreSQL) база данных создается командой:

CREATE DATABASE имя_базы_данных;

Пример: CREATE DATABASE company;

5. Как создать пользователя?

Пользователь создается командой:

CREATE USER 'имя_пользователя' IDENTIFIED BY 'пароль';

Пример: CREATE USER 'manager' IDENTIFIED BY 'securepassword123';

6. Почему необходимо разделение прав доступа в БД?

Разделение прав доступа необходимо для:

  • Безопасности: Ограничение доступа к конфиденциальным данным
  • Минимизации рисков: Предотвращение случайного или умышленного повреждения данных
  • Соблюдения принципа минимальных привилегий: Пользователь получает только те права, которые необходимы для выполнения его задач
  • Аудита и отчетности: Возможность отслеживать действия конкретных пользователей

7. Как дать пользователю необходимые права только на эту БД?

Права предоставляются командой GRANT с указанием конкретной базы данных:

GRANT SELECT, INSERT, UPDATE ON имя_базы_данных.* TO 'имя_пользователя';

Пример: GRANT SELECT, INSERT ON company.* TO 'manager';

После назначения прав необходимо выполнить:

FLUSH PRIVILEGES;

8. Что необходимо для безопасного хранения учетных данных?

Для безопасного хранения учетных данных необходимо:

  • Хеширование паролей: Использование алгоритмов bcrypt, Argon2, PBKDF2
  • Соль (salt): Добавление случайных данных к паролю перед хешированием
  • Защита от перебора (brute-force): Ограничение попыток входа, капча
  • Шифрование конфиденциальных данных: Использование шифрования для особо важных данных
  • Регулярное обновление паролей: Политика смены паролей
  • Использование защищенных каналов передачи: HTTPS, SSH
  • Аудит и мониторинг: Отслеживание подозрительной активности
